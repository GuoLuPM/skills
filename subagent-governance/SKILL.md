# 子代理提示里的逻辑与兜底校验

## 概览

这份技能只做一件事：
给研究型子代理的提示词补上“业务逻辑校验”或“降级 / 兜底校验”要求。

它不负责：

- 决定是否要起子代理
- 决定要起几个子代理
- 约束实现型或复核型子代理
- 限制主代理如何拆分其他任务

## 什么时候用

- 你已经准备派出研究型子代理。
- 你担心子代理只看局部代码，没有验证主链。
- 你担心子代理遗漏默认值补偿、兜底逻辑或降级链路。
- 你希望返回结果明确区分已确认事实和假设。

## 不适用

- 你还在判断要不要派子代理。
- 你要写的是实现型或复核型提示词。
- 任务只是简单查文件、搜文本、跑命令，不需要业务逻辑或兜底校验。

## 用法

在你原本的派发提示后，按需要追加下面其中一组要求。不要用它替换原来的任务目标。

### 业务逻辑校验

- 说明主路径、唯一真值、主收口点、消费链。
- 标出哪些模块只是透传、编排或消费。
- 每个关键结论都附文件路径与函数、符号证据。
- 不确定项标记为 `[假设]`。

### 降级 / 兜底校验

- 说明主路径、触发条件、降级路径、决策点。
- 说明进入降级后损失了什么语义或交互保证。
- 说明是否存在日志、告警或其他可观测信号。
- 每个关键结论都附文件路径与函数、符号证据。
- 不确定项标记为 `[假设]`。

## 组合规则

- 如果你只关心主链，就只加“业务逻辑校验”。
- 如果你只关心兜底，就只加“降级 / 兜底校验”。
- 如果两者都重要，可以两组都加。
- 两组都加，不等于必须多起一个子代理；是否拆成多个子代理，由主代理自己决定。

## 可直接追加的段落

```md
额外校验要求：
- 从业务逻辑视角补充说明主路径、唯一真值、主收口点、消费链。
- 同时检查是否存在降级 / 兜底链路，并说明触发条件、决策点、降级路径与语义损失。
- 每个关键结论都附文件路径与函数 / 符号证据。
- 不确定项标记为 [假设]。
```

## 约束

- 这份技能只补充“要验证什么”，不替你决定“要不要派”“派几个”“怎么并行”。
- 如果原任务与业务逻辑或兜底无关，不要硬套这份技能。
- 不要因为用了这份技能，就限制主代理起其他类型的子代理。
